启动方式

平台启动器
macOS双击 CodexPortable.command
Windows双击 CodexPortable.bat
Linux./CodexPortable.sh

首次运行 CC Switch GUI 自动打开，在 Codex 标签添加供应商保存 → Codex CLI 启动。

贰

API 配置

Codex CLI 通过两个文件管理配置：

  • data/.codex/auth.json — API Key
  • data/.codex/config.toml — 模型供应商定义

CC Switch 自动配置

启动后 CC Switch 切换到 Codex 标签，选择模板：

  • OpenAI 官方
  • 第三方中转（one-api / new-api / 等）
  • DeepSeek / Kimi / GLM 等国产模型
  • Azure OpenAI

手动配置示例

{"OPENAI_API_KEY": "sk-..."}
model_provider = "custom"
model = "gpt-5.4"
model_reasoning_effort = "high"

[model_providers.custom]
name = "Custom"
base_url = "https://your-provider/v1"
wire_api = "responses"
env_key = "OPENAI_API_KEY"
叁

设备绑定

首次成功运行后，便携包绑定到当前 U 盘的文件系统 UUID。复制到其他设备运行将报错。

解绑命令

# macOS
./CodexPortable.command --unlock

# Linux
./CodexPortable.sh --unlock

# Windows
CodexPortable.bat --unlock
💡 双 lock 文件镜像，删除单个不能绕过绑定
